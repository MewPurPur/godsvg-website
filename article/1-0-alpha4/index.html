New prerelease: GodSVG 1.0-alpha4

By: MewPurPur Jun 6, 2024
NOTE:
This blog post was written retroactively to document events from GodSVG's early development history.

Rework of attribute defaults

Right now, my biggest priority is to massively rework how SVGs are handled internally. I wasnâ€™t able to complete it for this prerelease, but I was able to salvage one small improvement from my attempts: A change to how default values are treated. Attribute fields will no longer have any text if the attribute hasn't been declared. Instead, they will show their default value as dimmed placeholder text.

Improved user experience

I worked with Qainguin to improve exporting. Web exports have been fixed, and support for WebP and JPEG formats has been added.

I've made popups display their keyboard shortcuts. This is pretty unobtrusive and allows for easier learning.

I've also made some speed optimizations for this release, which should make SVGs load ~30% faster on average. Some other changes should allow GodSVG to open much larger SVG files without crashing. Kiisu-Master and I also worked to fix some memory leaks.

Reference image

A common workflow in vector graphics is to vectorize over a background reference image. I worked with Qainguin to implement this feature for GodSVG on all Desktop platforms. The Web implementation will be a little more difficult, the technology is not there yet. It will be implemented at a later point.

Other goodies

  • Improvements to UI scale logic and settings (by aaronfranke and Kiisu-Master)
  • Debug information accessible through F3 (by me and Kiisu-Master)
  • "Show in List" functionality when right-clicking handles in the viewport (by WeaverSong)
  • More shortcuts for convenience, such as Ctrl+Q (by me)
  • Translation updates (by Vovkiv and JinEnMok)
  • Improvements to handling SVGs without specified dimensions (by me)
  • A lot of smaller improvements and bugfixes by me, Qainguin, ilikefrogs101, Kiisu-Master... too many to list. Maybe I'll start listing them when GodSVG becomes more stable.
